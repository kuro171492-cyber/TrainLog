# DECISIONS

## D-1: Переход на File System Access API
**Date**: 2026-03-31
**Status**: Active
**Context**: Нужна автоматическая синхронизация данных с файлом в репозитории
**Decision**: Использовать File System Access API (showOpenFilePicker/showSaveFilePicker) с IndexedDB для хранения handle
**Reason**: Позволяет читать/писать файл без ручного экспорта/импорта
**Tradeoffs**:
  + Автоматическое сохранение в trainlog_data.json
  + Интеграция с Git (изменения видны в diff)
  - Работает только в современных браузерах (Chromium)

## D-2: GitHub Sync как дополнительный слой
**Date**: 2026-03-31
**Status**: Active
**Context**: Нужна облачная синхронизация для кросс-девайс доступа
**Decision**: GitHub REST API (Contents endpoint) с PAT-аутентификацией
**Reason**: Бесплатно, не требует бэкенда, данные в Git-истории
**Tradeoffs**:
  + Бесплатно, не нужен свой сервер
  + Версионирование через Git
  - Требует PAT с правами Contents
  - Нет реального времени (только ручной upload/download)

## D-3: Vanilla JS без фреймворков
**Date**: 2026-03-31
**Status**: Active
**Context**: Проект должен работать без сборки и билд-системы
**Decision**: Чистый HTML/CSS/JS, все скрипты в глобальном скоупе
**Reason**: Ноль зависимостей при разработке, открывается из файла/gh-pages
**Tradeoffs**:
  + Нет билд-шага, простая архитектура
  + Работает из локального файла (file://)
  - Глобальные функции, сложнее тестировать
  - Код в одном скоупе без модулей

## D-4: Удаление аналитики и кнопок навигации
**Date**: 2026-05-25
**Status**: Active
**Context**: Приложение используется исключительно для логирования тренировок; аналитика не востребована, кнопки "Лог"/"Аналитика" создают лишний UI-шум
**Decision**: Удалён блок `#roomTable` (аналитика: графики Chart.js, таблица истории, фильтр), кнопки переключения "Лог"/"Аналитика", сервер-лаунчер
**Reason**: Упрощение интерфейса, уменьшение размера CSS/JS, устранение неиспользуемого кода
**Tradeoffs**:
  + Чистый UI без переключения вкладок
  + Меньше JS/CSS для загрузки
  - Потеря визуализации прогресса (графики, история)

## D-5: Mobile-first редизайн
**Date**: 2026-05-25
**Status**: Active
**Context**: Основная аудитория использует приложение на телефоне; десктоп — вторичен
**Decision**: Полный mobile-first редизайн: full-width карточки, нулевой border-radius у day-card на ≤640px, 16px радиус у exercise/superset, схлопывание пустых полей подходов, кнопка "+ Новая сессия" в навбар, красная тема (red-600) вместо синей
**Reason**: Улучшение UX на реальных устройствах, устранение скролл-скачков и проблем с кэшем
**Tradeoffs**:
  + Удобство на телефоне
  + Компактный интерфейс
  - Десктоп выглядит менее презентабельно (приоритет телефона)

## D-6: Реорганизация кнопок в заголовке (UI/UX улучшение)
**Date**: 2026-06-08
**Status**: Active
**Context**: Нужна более интуитивная структура кнопок в заголовке для мобильных устройств
**Decision**: 
  1. Кнопка "Новая сессия" перемещена на вторую строку заголовка и увеличена в 2 раза (px-6 py-2.5, h-6 w-6 иконка, text-sm)
  2. Кнопки файла (Экспорт, Импорт, Подключить) расположены слева в первой строке
  3. Кнопки GitHub (проверить, загрузить, сохранить) расположены справа в первой строке
**Reason**: Главное действие (добавить сессию) должно быть самым заметным; утилиты распределены логически (файлы слева, облако справа)
**Tradeoffs**:
  + Кнопка "Новая сессия" сразу заметна, удобнее кликать
  + Логическая группировка действий
  - Заголовок занимает немного больше места (две строки на мобильных)
